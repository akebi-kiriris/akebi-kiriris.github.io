# Personal Website 規劃筆記

## 1. 網站定位
這個網站的核心定位是：
- 作品集（Portfolio）
- 技術筆記（Markdown Notes）
- 成長軌跡（Learning Journey）

目標是讓訪客快速理解：
- 你做過什麼
- 你怎麼解問題
- 你正在往哪個方向成長

## 2. 可以放的內容（優先順序）

### 2.1 專案展示頁（最優先）
每個專案建議包含：
- 簡介（解決什麼問題）
- 技術棧
- 你負責的部分
- 成果截圖
- Repo / Demo 連結
- 反思（遇到什麼問題、如何解）

### 2.2 Markdown 筆記系統
把筆記分類管理，例如：
- Frontend
- Backend
- AI
- System Design
- Debug 記錄

每篇建議加上：
- 標籤
- 最後更新日期
- 3 行摘要

### 2.3 學習路線圖頁
可用 `Now / Next / Later` 呈現：
- Now: 正在學
- Next: 下一步要補
- Later: 中長期目標

### 2.4 實作日誌（Build in Public）
每週短記錄：
- 這週做了什麼
- 卡在哪裡
- 如何解決
- 下週要做什麼

### 2.5 About Me 頁
簡潔說明：
- 你是誰
- 你在解哪類問題
- 擅長領域
- 想合作的方向

## 3. 若要加後端，可做的小功能
- 全站搜尋（專案 + 筆記）
- 閱讀數或按讚
- 留言 / 回饋表單
- 電子報訂閱
- AI 助手（基於你網站內容的 Q&A）

## 4. 平台選擇建議（GitHub / Firebase）

### 4.1 GitHub Pages（先上線最快）
適合：
- 以靜態展示為主
- 維護簡單
- 成本低

### 4.2 Firebase Hosting + Firestore + Auth
適合：
- 需要互動功能
- 需要登入或資料儲存
- 後續可擴充動態功能

### 4.3 建議路線
- 第一步：先用靜態方式上線（快速有成果）
- 第二步：有需求再補 Firebase 後端功能

## 5. 建議先做的 MVP
1. 首頁（自我介紹 + 精選專案）
2. 專案列表頁
3. 筆記列表頁 + 文章頁（讀取 md）
4. 聯絡方式頁

---

## 6. 這種規劃檔通常放哪裡？
常見做法：
- `README.md`：放「對外簡介」與快速入口
- `docs/`：放「規劃文件、內容策略、架構設計」
- `notes/` 或 `planning/`：放個人草稿（團隊較少用）

這份文件屬於規劃/策略內容，最常見是放在：
- `docs/website-content-plan.md`

如果你只有一個文件，也可以先放根目錄；但中長期建議統一放 `docs/` 比較好管理。
