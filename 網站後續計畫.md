# Kiriris 網站後續計畫（純前端階段）

## 目前方向
- 短期不加入後端
- 維持純前端讀取 Markdown（`frontend/src/content/notes/*.md`）
- 先把內容體驗從「可讀」提升到「有層次、有記憶點」

## 目標
- 保留輕量維護優勢
- 減少單純文章展示的單調感
- 先把結構與閱讀流程打磨好，為未來後端/RAG 預留空間

## 短期優先事項

### 1. 筆記資料結構升級（frontmatter）
每篇筆記統一欄位：
- `title`
- `date`
- `tags`
- `summary`
- `level`
- `readingTime`
- `related`

預期效果：
- 列表資訊更完整（難度、閱讀時間）
- 篩選和推薦更自然

### 2. 筆記體驗升級（純前端）
- 文章目錄（TOC）
- 閱讀進度條
- 上一篇 / 下一篇
- 相關文章推薦（以 tags/related 為主）

預期效果：
- 減少跳出率
- 提升連續閱讀體驗

### 3. 專案頁內容模板化
每個專案固定結構：
- 問題
- 解法
- 取捨
- 結果
- 下一步

預期效果：
- 從「貼文章」變成「展示思考與能力」

## 建議執行順序
1. 先完成 frontmatter 規格與解析
2. 再做 TOC + 閱讀進度 + 相關文章
3. 最後補專案頁模板與現有專案內容

## 備註
- 後端功能（留言、分析、RAG）保留到下一階段
- 現階段以內容品質與資訊架構為主，不追求功能數量

## 下一階段功能優先級（純前端）
1. `Code Block 升級`：
   - 顯示語言標籤
   - 一鍵複製按鈕
   - 後續可加檔名、重點行、diff block
2. `MD Callout 系統`：
   - `Note / Warning / Architecture / Tradeoff / Insight`
   - 把筆記升級為更像工程文章的閱讀體驗
3. `Command Palette (Ctrl + K)`：
   - 搜尋筆記
   - 快速跳頁
   - Tag 快速篩選
4. `Timeline / Learning Journey`：
   - 專案 phase 演進
   - 架構調整紀錄
5. `Knowledge Graph（先迷你版）`：
   - related notes mini graph
   - tag/topic 關聯視圖

## 實作順序建議（目前版本）
1. 先完成 `Code Block 升級`
2. 再做 `MD Callout 系統`
3. 接著做 `Command Palette`
4. 最後補 `Timeline` 與 `Knowledge Graph mini`
