## 為什麼要測試

### 從「會寫」到「掌控」

| 階段 | 特徵 | 痛點 |
|------|------|------|
| **寫出來能動** | 反覆 F5 確認功能 ✅ | 無法量化品質，上線前膽戰心驚 |
| **寫出能維護的代碼** | 邏輯清晰、重構不怕  ✅ | 但重構時仍需手動測試整條流程 |
| **有測試的代碼** 🎯 | 自動驗證「改 A 沒破 B」 ✅ | 面試官眼中的「工業級」開發者 |

### 你的三個收穫

1. **代碼自信度飆升**：修改任何邏輯後，測試套件 1 秒內告訴你有沒有破東西
2. **重構膽量翻倍**：Service 層改寫、Store 邏輯優化都不再膽戰心驚
3. **履歷價值爆表**：「寫 200+ 測試用例」比「做 5 個新功能」對 Senior 職位的說服力更強

## 測試金字塔

```
        🔺 端到端（E2E）— 模擬真實用戶
       /  \   —— 5~10 個關鍵流程
      /    \
     /------\
    / 整合測試 \ —— Service + Store 互動
   /          \  —— 20~30 個測試案例
  /            \
 /──────────────\
/  單元測試      \ —— 函式 / 方法 / 組件
\________________/ —— 100+ 個測試案例（最多）
```

### PrAjeKt 採取的策略

| 層級 | 工具 | 覆蓋 | 舉例 |
|------|------|------|------|
| **單元測試** | Vitest (前端) + Pytest (後端) | 95% | Service 層各方法、Validator 工具函式 |
| **整合測試** | Vitest + Pytest | 80% | Store + Service 互動、路由權限檢查 |
| **E2E 測試** | Playwright / Cypress | 60% | 登入 → 建立任務 → 群組聊天（先不做） |

## 前端測試策略（Vitest）

### 為什麼選 Vitest？

- ⚡ 與 Vite 無縫整合（開發環境已有）
- 📝 Vue 3 + TypeScript 原生支援
- 🚀 出測結果快，反饋迴圈短
- 🎯 語法熟悉（Jest 兼容）

### 測試涵蓋範圍

```
前端測試類型                   優先級
├── Composable 邏輯          🔴 最高（useConfirm, useFormatDate）
├── Store (Pinia)            🔴 最高（todos, tasks, timelines）
├── Service 層（API 呼叫）    🔴 最高（todoService, taskService）
├── Formatter / Validator    🟡 高（formatters.ts, validators.ts）
├── 組件邏輯（不含樣式）     🟡 中（Header, Sidebar, Modal）
└── Complex UI 互動          🟢 低（拖曳、多步表單，先不做）
```

```typescript
// frontend/src/services/__tests__/todoService.test.ts
import { describe, it, expect, beforeEach, vi } from 'vitest';
import { todoService } from '../todoService';
import api from '../api';

// Mock axios
vi.mock('../api');

describe('todoService', () => {
  beforeEach(() => {
    vi.clearAllMocks();
  });

  describe('getAll', () => {
    it('應該返回所有待辦清單', async () => {
      const mockData = [
        { id: 1, title: 'Task 1', completed: false },
        { id: 2, title: 'Task 2', completed: true },
      ];
      
      vi.mocked(api.get).mockResolvedValue({ data: mockData });
      
      const result = await todoService.getAll();
      
      expect(api.get).toHaveBeenCalledWith('/todos');
      expect(result).toEqual(mockData);
      expect(result).toHaveLength(2);
    });

    it('API 呼叫失敗時應拋出錯誤', async () => {
      vi.mocked(api.get).mockRejectedValue(new Error('Network Error'));
      
      await expect(todoService.getAll()).rejects.toThrow('Network Error');
    });
  });

  describe('addTodo', () => {
    it('應該以正確的 payload 建立待辦', async () => {
      const newTodo = { title: 'New Task', content: 'Details' };
      const mockResponse = { id: 3, ...newTodo, completed: false };
      
      vi.mocked(api.post).mockResolvedValue({ data: mockResponse });
      
      const result = await todoService.addTodo(newTodo);
      
      expect(api.post).toHaveBeenCalledWith('/todos', 
        expect.objectContaining(newTodo)
      );
      expect(result.id).toBe(3);
    });
  });

  describe('toggleComplete', () => {
    it('應該切換待辦完成狀態', async () => {
      vi.mocked(api.patch).mockResolvedValue({ 
        data: { id: 1, completed: true } 
      });
      
      const result = await todoService.toggleComplete(1);
      
      expect(api.patch).toHaveBeenCalledWith('/todos/1/toggle');
      expect(result.completed).toBe(true);
    });
  });
});
```

## 後端測試策略（Pytest）

### 為什麼選 Pytest？

- 📝 Python 官方標準（Flask 推薦）
- 🔧 Fixture 系統強大（數據準備、數據庫隔離）
- 🎯 Mock 和 Patch 整合完美
- ⚡ 支援 Parametrize（一次測試多個案例）

### 測試涵蓋範圍

```
後端測試類型                        優先級
├── Service 層邏輯                 🔴 最高（權限檢查、資料轉換）
├── Route 端點 (Blueprint)         🔴 最高（HTTP 請求/回應）
├── Model 層（ORM）                🟡 高（序列化、驗證）
├── Decorator （權限）             🟡 高（身分驗證、角色檢驗）
├── 實際資料庫操作（整合測試）     🟡 中（Transaction、Cascade）
└── 複雜業務邏輯（Service）        🟢 中（通知、計算）
```

```python
# backend/tests/test_blueprints/test_tasks.py
import pytest
from flask import json

@pytest.fixture
def auth_headers(client, test_user):
    """建立授權請求頭"""
    response = client.post('/api/auth/login', json={
        'username': test_user['username'],
        'password': test_user['password']
    })
    token = response.json['access_token']
    return {'Authorization': f'Bearer {token}'}

class TestTasksAPI:
    """測試任務 API 端點"""
    
    def test_get_tasks_should_return_200(self, client, auth_headers):
        """取得任務清單應返回 200"""
        response = client.get('/api/tasks', headers=auth_headers)
        assert response.status_code == 200
        assert isinstance(response.json, list)
    
    def test_create_task_with_invalid_data_should_return_400(
        self, client, auth_headers
    ):
        """無效資料應返回 400 Bad Request"""
        response = client.post(
            '/api/tasks',
            headers=auth_headers,
            json={'name': ''}  # 名稱不能為空
        )
        assert response.status_code == 400
    
    def test_create_task_should_auto_assign_owner(
        self, client, auth_headers, db_session
    ):
        """建立任務應自動將建立者設為負責人"""
        response = client.post(
            '/api/tasks',
            headers=auth_headers,
            json={
                'name': 'New Task',
                'timeline_id': 1
            }
        )
        assert response.status_code == 201
        
        task_id = response.json['id']
        task = db_session.query(Task).get(task_id)
        
        # 檢查 task_user 記錄
        task_user = db_session.query(TaskUser).filter(
            TaskUser.task_id == task_id,
            TaskUser.role == 0  # Owner
        ).first()
        assert task_user is not None
    
    def test_only_owner_can_delete_task(
        self, client, auth_headers, test_task
    ):
        """只有負責人能刪除任務"""
        # 以負責人身分刪除 - 應成功
        response = client.delete(
            f'/api/tasks/{test_task.id}',
            headers=auth_headers
        )
        assert response.status_code == 204
        
        # 驗證任務已被軟刪除
        task = db_session.query(Task).get(test_task.id)
        assert task.deleted_at is not None
    
    def test_non_owner_cannot_delete_task(
        self, client, auth_headers, test_task, other_user_headers
    ):
        """非負責人不能刪除任務"""
        response = client.delete(
            f'/api/tasks/{test_task.id}',
            headers=other_user_headers
        )
        assert response.status_code == 403
```

```python
# backend/tests/test_decorators/test_require_task_role.py
import pytest
from decorators import require_task_role

class TestRequireTaskRole:
    """測試任務角色權限檢驗"""
    
    def test_owner_should_pass_owner_check(
        self, client, auth_headers, test_task
    ):
        """負責人應通過 owner 檢查"""
        # 建立一個受保護的端點
        @require_task_role('owner')
        def protected_route(task_id):
            return {'status': 'ok'}
        
        # 負責人呼叫應成功
        response = client.get(
            f'/api/tasks/{test_task.id}/protected',
            headers=auth_headers
        )
        assert response.status_code == 200
    
    def test_member_should_fail_owner_check(
        self, client, other_user_headers, test_task, db_session
    ):
        """成員應無法通過 owner 檢查"""
        # 將 other_user 設為成員（role=1）
        task_user = TaskUser(
            task_id=test_task.id,
            user_id=other_user['id'],
            role=1  # Collaborator
        )
        db_session.add(task_user)
        db_session.commit()
        
        response = client.get(
            f'/api/tasks/{test_task.id}/protected',
            headers=other_user_headers
        )
        assert response.status_code == 403
    
    def test_non_member_should_be_rejected(
        self, client, stranger_headers, test_task
    ):
        """非成員應被拒絕"""
        response = client.get(
            f'/api/tasks/{test_task.id}/protected',
            headers=stranger_headers
        )
        assert response.status_code == 403
```

```python
import pytest
```

## 納入 CI/CD 流程

### GitHub Actions 設定

```yaml
# .github/workflows/test.yml
name: Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  frontend-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          cache-dependency-path: frontend/package-lock.json
      
      - name: Install dependencies
        working-directory: frontend
        run: npm ci
      
      - name: Run tests
        working-directory: frontend
        run: npm run test
      
      - name: Upload coverage
        if: always()
        uses: codecov/codecov-action@v3
        with:
          files: ./frontend/coverage/coverage-final.json
  
  backend-tests:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_PASSWORD: postgres
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'
          cache: 'pip'
      
      - name: Install dependencies
        working-directory: backend
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
          pip install pytest pytest-cov pytest-flask pytest-mock
      
      - name: Run tests
        working-directory: backend
        env:
          DATABASE_URL: postgresql://postgres:postgres@localhost/test_db
        run: pytest --cov=blueprint --cov=services --cov=models
      
      - name: Upload coverage
        if: always()
        uses: codecov/codecov-action@v3
        with:
          files: ./backend/coverage.xml

  build:
    needs: [frontend-tests, backend-tests]
    runs-on: ubuntu-latest
    if: success()
    steps:
      - uses: actions/checkout@v4
      
      - name: Deploy to production
        # 只在 main 分支且所有測試都通過時部署
        if: github.ref == 'refs/heads/main'
        run: |
          echo "Deploying to Railway..."
          # Railway 自動部署已配置
```

### 本地開發工作流

```bash
# 開發時實時執行測試
cd frontend && npm run test -- --watch

# 執行特定測試
npm run test -- todoService.test.ts

# 生成覆蓋率報告
npm run test:coverage

# 後端
cd backend
pytest --watch  # 需要 pytest-watch
pytest tests/test_services/test_task_service.py -v  # 詳細輸出
pytest --cov=services --cov-report=html  # HTML 覆蓋率報告
```
