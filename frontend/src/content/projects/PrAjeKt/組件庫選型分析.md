# PrAjeKt 組件庫選型分析

> 建立時間：2026/03/02  
> 背景：Phase 2 開始前決定 UI 組件策略，主要需求為通知系統（Toast）、確認對話框（Dialog）、數據圖表

---

## 現有 Stack 關鍵點

```
Vue 3.5 + Vite 7
Tailwind CSS v4（重要：非 v3，CSS-first 設定，無 tailwind.config.js）
Pinia + Vue Router + Axios
FullCalendar、vuedraggable（已安裝）
```

**Tailwind v4 是本次選型的核心限制因素**，  
許多主流方案都預設 Tailwind v3，需特別注意相容性。

---

## 方案一：Element Plus

**類型：** 完整 UI Kit（有自己的設計系統）

### 優點
- Vue 3 官方推薦，文件完整，中文友善
- 元件齊全：Table、Form、Dialog、Notification、Upload 全都有
- 不需要什麼都自己刻，開發速度快

### 缺點
- **和 Tailwind 嚴重衝突** — 有自己的 CSS 變數和重置規則，兩套 design token 並存會很亂
- 引入後套件體積大（即使 tree-shaking 也有一定基底）
- 外觀是 Element 風格，與現有 Tailwind 客製化頁面風格不統一
- 要「改造」Element Plus 的外觀成本非常高

### 結論
❌ **不建議**。你已有 Tailwind 且有現成樣式，Element Plus 進來會打架。

---

## 方案二：Naive UI

**類型：** 完整 UI Kit，TypeScript-first，主題可高度客製

### 優點
- 純 Vue 3 + TypeScript，設計相對現代
- 主題系統比 Element Plus 更彈性（可以用 CSS 變數覆蓋）
- 文件有中文版
- 圖表部分與 ECharts 整合有現成範例

### 缺點
- **同樣有自己的設計系統**，和 Tailwind 並用仍會有衝突
- 現在是純 JS 專案，Naive UI 的 TypeScript 優勢暫時用不到
- 套件體積大（~800KB minified）

### 結論
❌ **不建議**。理由同 Element Plus，兩套設計系統並存在同一個 SPA 成本大於收益。

---

## 方案三：Headless UI（官方 Vue 版）

**套件：** `@headlessui/vue`  
**官網：** headlessui.com

**類型：** Headless（完全無樣式），Tailwind 官方出品

### 優點
- Tailwind Labs 官方維護，100% 設計給 Tailwind 使用
- 完全無樣式，外觀完全由你的 Tailwind 類別控制，風格統一無衝突
- 提供無障礙（a11y）邏輯：鍵盤導航、focus trap、ARIA 屬性都處理好了
- 套件極輕量（僅邏輯層）
- **Tailwind v4 完全相容**（同家產品）

### 缺點
- **元件數量少**：僅提供 Menu、Listbox、Combobox、Switch、Dialog、Disclosure、Popover、RadioGroup、Tabs、Transition
- **沒有 Toast / Notification 元件** ← 這是最大缺口
- 需要自己寫大量樣式（但你已經在做這件事了）

### 結論
⚠️ **部分推薦**。Dialog / Dropdown / Tabs 這些用 Headless UI 很合適，但 Toast 系統要另外找方案。

---

## 方案四：Reka UI（原 Radix Vue）

**套件：** `reka-ui`  
**官網：** reka-ui.com

**類型：** Headless，Vue 3 原生，比 Headless UI 元件更多

### 優點
- 元件數量遠比 Headless UI 多（含 Toast/Sonner、Tooltip、DatePicker 等）
- 完全 Headless，和 Tailwind 零衝突
- 高度無障礙合規（基於 WAI-ARIA 規範）
- 社群活躍，持續更新
- shadcn-vue 的底層就是 Reka UI

### 缺點
- 文件相對 Element Plus 較少中文資源
- 比 Headless UI 設定稍微複雜一點
- Tailwind v4 相容性：本身無問題，但搭配 shadcn-vue 時有 v4 限制（見方案五）

### 結論
✅ **可考慮**。如果要 Headless 路線但需要更多元件，Reka UI 是升級版 Headless UI。

---

## 方案五：shadcn-vue

**套件：** 非安裝式，透過 CLI 複製元件原始碼到專案中  
**官網：** shadcn-vue.com  
**底層：** Reka UI + Tailwind + class-variance-authority (cva)

**類型：** 「複製式」UI 套件（你擁有元件原始碼，可直接修改）

### 優點
- 每個元件都是你的程式碼，完全可客製
- 設計精美，現代感強（類似 Vercel/Linear 風格）
- 包含 Toast（基於 vue-sonner）、Dialog、Table、DatePicker 等完整元件
- 不是黑箱，不怕某個版本出問題

### 缺點
- ⚠️ **Tailwind v4 相容性問題**：shadcn-vue 官方對 v4 的支援在 2026/03 仍不完整（還在 open issue 狀態），預設設定假設 v3
- 需要 `tailwind.config.js` 和特定的 CSS 變數設定，與 v4 的 CSS-first 方式不同
- 對 CSS 變數和 `cn()` 工具函式有額外依賴

### 結論
⚠️ **目前不建議**（Tailwind v4 相容性問題尚未解決，貿然引入會踩坑）。等 shadcn-vue 官方完整支援 v4 後再評估。

---

## 方案六：專用小套件（最輕量路線）

不引入整套 UI 框架，只針對需求引入對應小套件：

| 需求 | 套件 | 大小 | 說明 |
|------|------|------|------|
| Toast 通知 | `vue-sonner` | ~10KB | 精美的堆疊式 Toast |
| Toast（備選） | `vue-toastification` | ~15KB | 功能完整，高度客製化 |
| Dialog / Modal | 自製 + `@headlessui/vue` | 極輕量 | 你的 Modal 已有基礎 |
| 圖表 | `chart.js` + `vue-chartjs` | ~200KB | 簡單易用，文件多 |
| 圖表（備選） | `echarts` + `vue-echarts` | ~900KB | 功能強大，但較重 |
| 進階 Dropdown 等 | `@headlessui/vue` | ~30KB | Tailwind 官方出品 |

### 結論
✅ **最推薦的輕量路線**，保持現有 Tailwind 風格不變，只補缺口。

## 綜合比較表

| 方案 | Tailwind v4 相容 | 風格衝突 | 套件大小 | 上手難度 | 元件完整度 |
|------|:---:|:---:|:---:|:---:|:---:|
| Element Plus | ❌ 會衝突 | 高 | 大 | 低 | 非常完整 |
| Naive UI | ⚠️ 可用但衝突 | 高 | 大 | 低 | 非常完整 |
| Headless UI | ✅ | 無 | 極小 | 中 | 少（無 Toast）|
| Reka UI | ✅ | 無 | 小 | 中高 | 完整 |

| 專用小套件 | ✅ | 無 | 極小 | 低 | 按需安裝 |

## 推薦方向

### 建議採用：**Headless UI + vue-sonner + ECharts**

```
@headlessui/vue   → Dialog 確認框、Dropdown、Transition 動畫
vue-sonner        → Toast 通知系統（Phase 2 第一個需求）
echarts           → 數據分析儀表板（Phase 2 第二個需求）
vue-echarts       → ECharts 的 Vue 3 wrapper
```

**理由：**
1. **零風格衝突** — 三個套件都不帶自己的樣式，100% Tailwind 控制
2. **Tailwind v4 全相容** — 沒有 config 依賴問題
3. **剛好補足缺口** — 你現在缺的剛好就是 Toast 和 Chart，其餘 Modal 已有
4. **長期可擴展** — Phase 4 甘特圖、熱力圖直接用同一套，不需換庫
5. **不破壞現有風格** — 現在頁面的 Tailwind 樣式完全保留

### 安裝指令（待討論確認後執行）

```bash
cd frontend

# Headless UI（Dialog、Dropdown、Transition 等）
npm install @headlessui/vue

# Toast 系統
npm install vue-sonner

# 圖表（Phase 2 數據分析用）
npm install echarts vue-echarts
```

## Phase 2 對應關係

| Phase 2 功能 | 使用的新套件 | 備註 |
|------|------|------|
| 通知系統 Toast | `vue-sonner` | 替換全站 `alert()` |
| 確認刪除 Dialog | `@headlessui/vue` Dialog | 替換全站 `confirm()` |
| 數據分析圖表 | `echarts` + `vue-echarts` | 完成率、逾期率，Phase 4 甘特圖亦可沿用 |
| Dropdown Menu | `@headlessui/vue` Menu | Header 用戶選單 |
